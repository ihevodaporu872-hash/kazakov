### Конвертация единиц (функция `toWatts`)

```
if unit === 'gcal' → val × 1 163 000
if unit === 'mw'   → val × 1 000 000
if unit === 'kw'   → val × 1 000
```

### Расчёт Дельта-T (функция `calcDeltaT`)

```
Δt = (t_подачи + t_обратки) / 2 − t_внутр
```

Пример: график 80/60, t_внутр = 20 --> Δt = (80 + 60) / 2 - 20 = 50°C.

### Зависимости между параметрами

- `windows[]` -- массив окон, каждое: `{ width_mm, deviceLength_mm }`
- `deviceLength_mm = width_mm - 200` (длина прибора = ширина окна минус 200 мм)
- `powerPerMeter_W` -- мощность на 1 п.м. прибора (рассчитывается в Шаге 2)

## Блок 2: Подбор приборов отопления

### 2.1. Расчёт удельной мощности Q_м

Формула из `Step2_Windows`:

```
Q_м = Q_общ / (Σ_ширин_окон_м − N × 0.2)
```

Где:
- `Q_общ` -- общий расход тепла (Вт)
- `Σ_ширин_окон_м` -- сумма ширин окон в метрах
- `N` -- количество окон
- `0.2` -- отступ прибора от края окна (по 100 мм с каждой стороны)

Результат `Q_м` (Вт/м) -- требуемая мощность на 1 погонный метр прибора.

### 2.2. Длина прибора

```
deviceLength_mm = ширина_окна_мм − 200
```

Для панельных радиаторов длина округляется вниз до ближайшего стандартного размера (функция `nearestStdLengthDown`).

Стандартные длины (мм):

```
400, 500, 600, 700, 800, 900, 1000, 1100, 1200,
1400, 1600, 1800, 2000, 2300, 2600, 3000
```

### 2.3. Высота прибора

Определяется типом прибора (функция в `Step3_DeviceType`):

| Тип | Высота |
|---|---|
| Внутрипольные конвекторы | Фиксированная: **150 мм** |
| Напольные радиаторы/конвекторы | `простенок − стяжка − 200 мм` (мин. 300 мм) |
| Настенные радиаторы | `простенок − стяжка − 200 мм` (мин. 300 мм) |

```
if deviceType === 'inFloor':
    height = 150
else:
    height = wallHeight_mm − screedHeight_mm − 200
    if height < 200: height = 300
```

### 2.4. Пересчёт мощности по EN 442

Каталожная мощность приводится при стандартном Δt = 50°C. Пересчёт на фактический Δt:

```
Q_факт = Q_кат × (Δt_факт / 50) ^ 1.3
```

Функция `correctPower(qCat, dtCat, dtFact)`:

```javascript
return qCat * Math.pow(dtFact / dtCat, 1.3);
```

### 2.5. Алгоритм подбора панельных радиаторов

Порядок перебора типов панелей:

```
11 → 21s → 22 → 33
```

Для каждого окна при фиксированной единой высоте:

1. Берём `stdLen = nearestStdLengthDown(deviceLength_mm)`
2. Для каждого типа панели (от тонкого к толстому):
   - Ищем в каталоге: `catalog[panelType][height][stdLen]` -> `Q_кат`
   - Пересчитываем: `Q_факт = correctPower(Q_кат, 50, Δt)`
   - Если `Q_факт >= Q_треб × 0.95` (допуск **−5%**) -- прибор подходит, берём этот тип
3. Если ни один тип не подходит -- берём самый мощный (тип 33)

Результат подбора содержит: тип панели, высоту, длину, Q_кат, Q_факт, запас (%).

### 2.6. Алгоритм подбора внутрипольных конвекторов

Для внутрипольных конвекторов подбор идёт по **удельной мощности (Вт/м)** при выбранной глубине:

1. Выбор глубины: 90, 110, 130, 180, 240, 300 мм
2. Выбор типа: без вентилятора (`natural`) или с вентилятором (`fan`)
3. Из каталога: `catalogs.convectors[глубина][тип]` -> `Q_кат_Вт_м` (при Δt=50°C)
4. Пересчёт: `Q_факт_Вт_м = correctPower(Q_кат_Вт_м, 50, Δt)`
5. Проверка: `Q_факт_Вт_м >= powerPerMeter_W` -- достаточно ли

Для каждого окна:
```
Q_треб = powerPerMeter_W × (deviceLength_mm / 1000)
Q_факт = Q_факт_Вт_м × (deviceLength_mm / 1000)
```

### 2.7. findBestUnifiedHeight -- единая высота для эстетики

Цель: подобрать одну высоту для всех приборов в здании, чтобы минимизировать использование толстых типов (33, 22).

Алгоритм:

1. Собрать все доступные высоты из каталога (все типы, <= maxHeight)
2. Для каждой высоты `h`:
   - Для каждого окна определить минимальный тип панели, дающий >= 95% требуемой мощности
   - Запомнить максимальный индекс типа (`maxTypeIdx`) и число отказов (`failCount`)
   - Оценка: `score = maxTypeIdx × 100 + failCount × 1000`
3. Выбрать высоту с минимальной оценкой (тоньше типы + меньше отказов = лучше)

## Блок 5: Стальные стояки

Файл: `materialCalc.js`, раздел "БЛОК 5: СТАЛЬНЫЕ ТРУБОПРОВОДЫ".

### 5.1. Формула диаметра стояка

```
d = 18.8 × sqrt((0.86 × Q) / (ΔT × v))
```

Где:
- `Q` -- тепловая нагрузка на один стояк (кВт)
- `ΔT` -- разница температур подачи и обратки (°C), например для 80/60 -> ΔT = 20
- `v` -- скорость теплоносителя (м/с), по умолч. 0.7
- Результат `d` в мм, округляется вверх до стандартного DN

Тепловая нагрузка на стояк:

```
Q_стояк = Q_общ_кВт / (riserPairs × heatingZones)
```

Стандартные DN: 15, 20, 25, 32, 40, 50, 65, 80, 100.

### 5.2. Длина стояков

```
totalRiserLen = (floorHeight_м) × numFloors × riserPairs × 2 × heatingZones
```

Где множитель `2` -- подача + обратка (каждая пара стояков = 2 трубы).

### 5.3. Компенсаторы

Сильфонные осевые компенсаторы устанавливаются **через каждые 4 этажа**:

```
компенсаторов_на_стояк = ceil(numFloors / 4) − 1

totalCompensators = компенсаторов_на_стояк × riserPairs × 2 × heatingZones
```

При количестве этажей <= 4 компенсаторы не нужны.

### 5.4. Неподвижные опоры (НО)

```
НО_на_стояк = компенсаторов_на_стояк + 1

totalFixedSupports = НО_на_стояк × riserPairs × 2 × heatingZones
```

НО устанавливаются только при наличии компенсаторов (если компенсаторов = 0, то НО = 0).

### 5.5. Краны шаровые на стояках

По 2 на каждый стояк (верх + низ):

```
ballValves = riserPairs × 2 × 2 × heatingZones
```

### 5.6. Балансировочные клапаны

По 1 на каждую пару (подача+обратка):

```
balanceValves = riserPairs × 2 × heatingZones
```

### 5.7. Гильзы проходные и пена п/п

При проходе через перекрытия:

```
fireSleeves = riserPairs × 2 × heatingZones × numFloors
```

Диаметр гильзы = `riserDN + 20` мм.

Противопожарная пена: 1 баллон на ~20 проходов:

```
foamCans = ceil(fireSleeves / 20)
```

### 5.8. Грунтовка и покраска

```
грунтовка_кг = totalRiserLen × 0.1  (кг/м.п.)
эмаль_кг     = totalRiserLen × 0.15 (кг/м.п.)
```

### 5.9. Воздухоотводчики и спускники

| Позиция | Расположение | Кол-во |
|---|---|---|
| Воздухоотводчик автоматический | Верх стояков | riserPairs x 2 x heatingZones |
| Кран спускной (дренажный) DN15 | Низ стояков | riserPairs x 2 x heatingZones |

## Блок 6: Этажные коллекторы (гребёнки)

Файл: `materialCalc.js`, раздел "БЛОК 6".

### 6.1. Количество коллекторов

```
totalManifolds = manifoldOutputs × numFloors
```

### 6.2. Диаметр коллектора

Определяется по количеству отводов:

| Отводов | DN коллектора |
|---|---|
| <= 4 | DN32 |
| 5-8 | DN40 |
| > 8 | DN50 |

### 6.3. Краны и тройники

На каждый коллектор:

```
краны_шаровые = totalManifolds × 2
тройники      = totalManifolds × 2
```

### 6.4. Присоединительные фитинги

Для подключения PEX Ду20 к коллектору:

```
connFittings = totalManifolds × manifoldOutputs × 2  (подача + обратка)
```

## Блок 7: Горизонтальная разводка PEX

Файл: `materialCalc.js`, раздел "БЛОК 7".

Два варианта разводки: **лучевая** (radial) и **попутная** (series).

### 7.1. Лучевая разводка (через внутриквартирные коллекторы)

#### Внутриквартирные коллекторы

Количество отводов в квартире:

```
outputsPerApt = roomsPerApt + 1  (комнаты + кухня)
```

Количество коллекторов:

```
кол-во = apartments × 2  (подача + обратка)
```

Фитинги перехода PEX -> коллектор (евроконус):

```
pexToCollFittings = apartments × outputsPerApt × 2
```

#### Трубы PEX (лучевая)

| Диаметр | Назначение | Формула длины |
|---|---|---|
| PEX d16 | Подводка к приборам | `windowCount × 3` м |
| PEX d20 | От этажного колл. до квартирного | `apartments × 5` м |
| PEX d25 | Магистраль этажная | `corridorLen × numFloors × 2` или `manifoldOutputs × numFloors × 3` |

### 7.2. Попутная разводка (через тройники)

#### Тройники

Общее количество:

```
teeCount = (windowCount − apartments) × 2
```

Распределение по типоразмерам:

| Тройник | Назначение | Кол-во |
|---|---|---|
| PEX 25x16x20 | 1-й тройник от магистрали | min(apartments x 2, teeCount) |
| PEX 20x16x16 | 2-й тройник | min(apartments x 2, остаток) |
| PEX 16x16x16 | Остальные | оставшиеся |

#### Трубы PEX (попутная)

| Диаметр | Назначение | Формула длины |
|---|---|---|
| PEX d16 | Подводка к приборам | `windowCount × 1.5` м |
| PEX d20 | Разводка между приборами | `windowCount × 2.0` м |
| PEX d25 | Магистраль | `corridorLen × numFloors × 2` или `manifoldOutputs × numFloors × 3` |

### 7.3. Общая длина PEX

```
totalPexLen = pexD16 + pexD20 + pexD25
```

Все длины могут быть переопределены вручную через `state.pipeLenByDiam`.

## Блок 8: Дополнительные материалы

Файл: `materialCalc.js`, раздел "БЛОК 8".

### 8.1. Гофра для PEX внутри квартир

```
gofraPexLen = totalPexLen × 0.6   (60% от общей длины PEX)
```

### 8.2. Теплоизоляция ВПЭ

Для PEX в коридорах (межквартирное пространство):

```
insulPexLen = totalPexLen × 0.4   (40% от общей длины PEX)
```

Для стальных стояков:

```
insulRiserLen = totalRiserLen      (100% длины стояков)
```

Толщина изоляции: 9 мм или 13 мм (из параметров проекта).

### 8.3. Хомуты для крепления PEX к полу

```
pexClamps = totalPexLen × 2       (2 шт/м.п.)
```

### 8.4. Соединительные муфты PEX

Через каждую бухту (по умолч. 200 м):

```
pexCouplings = floor(totalPexLen / buhtaLen)
```

### 8.5. Напрессовочные гильзы PEX

Материал: пластик или металл (выбирается в параметрах).

```
pressConnections = windowCount × 2 + pexCouplings
```

Где `windowCount × 2` -- подача + обратка на каждый прибор, `pexCouplings` -- на муфтах.

Цена:
- Пластик: 25 руб/шт
- Металл: 45 руб/шт

### 8.6. Крепление стальных трубопроводов

Комплект (хомут + анкер + шпилька):

```
steelClamps = round(totalRiserLen / 1.5)   (~1 компл. на 1.5 м.п.)
```

### 8.7. Теплосчётчики

Количество задаётся вручную (`heatMeterCount`), для офисных/БКТ помещений.

### 8.8. ПНР: пусконаладочные работы

Общая длина трубопроводов для расчёта ПНР:

```
totalPipeLen = totalRiserLen + totalPexLen
```

| Позиция | Ед. изм. | Кол-во |
|---|---|---|
| Гидравлическое испытание системы | м.п. | totalPipeLen |
| Промывка системы отопления | м.п. | totalPipeLen |
| Опробование системы отопления | м.п. | totalPipeLen |
| Сдача системы отопления | компл. | 1 |

## Приложение: Ценообразование

Три уровня определения цены материала (функция `getMaterialPrice`):

1. **Рыночная матрица** (`price_matrix.json`) -- по ключу `priceKey`
2. **Бренд** (`brands.json`) -- по категории и выбранному бренду
3. **Фиксированная оценка** (`fixedPrice`) -- если нет данных в матрице/брендах

Для работ: цены берутся у выбранного подрядчика, при отсутствии -- у соседнего подрядчика, затем из рыночной матрицы.
